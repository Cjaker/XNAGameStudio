Introduction to Minjie

Minjie is a complete XNA Game Studio minigame. The project comes ready to compile and run. It is easy to customize with a little bit of C# programming. You are free to use the source code as the basis for your own game projects, and to share your work with others.

The goal in Minjie is to flip your opponent's pieces to your own color by surrounding them on two sides with your own pieces. For example, you can surround a line of pieces by first placing your piece on one end of the line. Then, on your next turn, place another piece at the other end of the line. All of the opponent's pieces that you surround will then flip over to your color. The play alternates with each player placing a piece at a location that causes a flip. The game ends when there are no more moves available for either player. The winner is the player with the most pieces on the board when the game ends. Minjie supports one player playing against the AI or two players playing against each other.

Getting Started

Building, and Running a Minjie Project

To build and run the Minjie project

  • Either press F5 or, on the Debug menu, click Start Debugging.

    The project will build and then run within the debugger.

To build the project without running it

  • Either press F6 or, on the Build menu, click Build Solution.

    The project will build without running.

Implementation Notes

Game Loop

MinjieGame.cs contains the main game loop. The MinjieGame class contains all the update code for each frame of the game loop.

Game Logic

Board.cs contains most of the game logic. The Board class contains all the functions needed to make moves and determine the winner. You can change the board size simply by changing the GameplayScreen.boardSize constant.

Graphics and Content

This game contains six models that are able to display all available game states. The tile used for the board has two models: normal and highlighted. You can use the highlighted model for the selected square, which is the normal model for the rest of the board. The entire board is stitched together using a combination of those two models. The game piece also has two models, a free standing model and an "on the board" model. All lighting and texture is displayed using basic effects.

Game State Management

This sample uses the infrastructure provided by the Game State Management sample, including the InputState input-management class.

The cursor moves in board-spaces. The rotation of the board determines the direction of movement. This guarantees consistent movement regardless of rotation.

Artificial Intelligence

The computer player typically uses a minimax algorithm to pick out the best move. The heuristic is the difference between the number of pieces of each player. For example, in the first layer of the algorithm, the move that maximizes the AI player's own number of pieces will be chosen. In the second layer, it is assumed that the opponent will chose the move that will minimize the AI player's number of pieces. In the third layer, the move that will again maximize the AI player's number of pieces will be chosen. Eventually, the layer depth will be met and the best move is reported back. The layer depth and node number being searched can be modified in AiPlayer.cs See the general discussion of minimax for a more detailed explanation.

Extending Minjie

Minjie's implementation is very simple. There are many aspects of the game that can easily be modified and extended. Some possibilities include the following:

Gameplay and Functionality

  • Support a network game, likely starting with the Network Game State Management sample and a new NetworkPlayer class mirroring the LocalPlayer and AiPlayer classes.
  • Keep score and display it with text or sprites.
  • Implement save and load game.
  • Allow replay of saved or current game.
  • Implement free controller movement that is not restricted to the board.
  • Extend the Board class to implement an entirely different game.
  • Create timed variations or implement tournament rules.
  • Add alpha-beta pruning to the minimax algorithm that Minjie uses.

Graphics

  • Create your own game pieces.
  • Animate the flipping of game pieces after a move, rather than the instant update used presently.
  • Implement shadows using directional lighting.
  • Add exciting and colorful custom effects that use these files. For example, the content folder includes the reflection maps for both the red and the green game pieces.

Minjie demonstrates simple 3D rendering, gameplay, and the minimax algorithm. It could serve as a quick start to your own strategy board game!

Introduction to Catapult

Catapult is a complete XNA Game Studio mini game. The project comes ready to compile and run, and it's easy to customize with a little bit of C# programming. You are free to use the source code as the basis for your own XNA Game Studio game projects, and to share your work with others.

The goal in Catapult is to send the pumpkin flying as far as possible. You do this by moving the catapult towards the log and launching the pumpkin. If you wait too long, the catapult smashes into the log and you have to try again. The all-time high score is kept in the upper-left corner. There are two ways to get boost power to help send your pumpkin flying. The first is to press the launch button close to the log. There are three levels of boost for launching close to the log. The boost level is shown by the number of exclamation points after the boost display is shown in the center of the screen shortly after launch. The other way to get boost power is to hold the right trigger on the gamepad at 50 percent or by pressing the B key on the keyboard.

Getting Started

Building and Running a Catapult Project

To build and run the Catapult project

  1. Press F5, or, on the Debug menu, click Start Debugging.

    The project will build and then run within the debugger.

To build the project without running it

  1. Press F6, or, on the Build menu, click Build Solution.

    The project will build without running.

Game Controls

Catapult uses the following keyboard and gamepad controls.

Action Keyboard Control Gamepad Control
Move the catapult. RIGHT ARROW Left trigger
Boost power. B Right trigger
Launch the pumpkin, and reset the catapult. SPACEBAR A
Exit the game. ESC BACK

Implementation Notes

Game Loop

The main game loop is in Game.cs. The CatapultGame class contains all the update code for each frame of the game loop. The CatapultGame class contains a Catapult object that is used to represent the catapult and the pumpkin.

Game Logic

Most of the game logic is in the Catapult class. The Catapult object handles its own input, update, and drawing. It also maintains its own state.

Graphics and Content

The game uses eight custom graphics, three of which represent the front, back, and arm of the catapult, respectively. Two graphics represent the pumpkin, in its while state and its broken state. One graphic is used for the foreground, and one is used for the distant background mountains. The last graphic is the log used to represent the end of the launch area for the catapult. All of the sounds come from previous XNA Game Studio Express starter kits.

Input

The game supports controller and keyboard input. However, since the keyboard does not support analog input, the keyboard's boost button (B) always gives the player the best possible boost.

Extending Catapult

Catapult's implementation is very simple, and there are many aspects of the game that can easily be modified and extended. Here are some possibilities with respect to gameplay and functionality.

  • Add obstacles that the pumpkin can hit to slow it down.
  • Add bonuses that the pumpkin can hit to add distance to the throw.
  • Add a high score table with names.
  • Change the pumpkin to a different projectile.
